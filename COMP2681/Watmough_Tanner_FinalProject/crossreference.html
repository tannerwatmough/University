Minimum Requirement Features

HTML5

  1. The header and footer on this page use the approrpiate structural tags. The main and section tags are also used. Finally, this page utilizes an <ol> element. Approrpiate HTML5 tags, including use of HTML5 structural tags, <a>, <p>, <blockquote>, <ol>, <ul>, and <dl>, to name a few.
  2. The links bring you to a clickable photo (the main logo which brings you to the home page) and the gallery which contains a few static images. Inline images both static and clickable
  3. The footer uses the special character • (∧#8226;) Use of approrpiate special characters
  4. The meta tags for the main page are
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta name="keywords" content="photography, ferelden, photographer" />     Use of approrpiate <meta> tags
  5. You can see the footer on each page, but this links you to the home page footer. You can also see the last modified date on the footer. A footer on the home page indicating when the website was last updated or modified

CSS3

  1. The audio paragraph on the home page is edited with a drop cap and the header has styles applied to it (as does the booking headers), the services page has lists that are modified by CSS, all pages, at sufficient size, show a background image behind the page body, the booking page as a table (calendar) with styling, and the contact page has a form with styling. Use CSS3 to format the appearance of paragraphs, lists, headers, background images, tables, and forms
  2. The reset sheet uses the e1, e2, e3 selectors. The navigation menu uses the e f selector, we use body > header > image to select the header image, and the mobile navicon menu uses e + f. Finally, the contact form uses the input[type] selector and the print sheet selects the href for link printing. Use CSS3 contextual selectors (e1, e2, e3; e f; e > f; e + f) and at least two attribute selectors, excluding class and id attribute selectors
  3. The navigation list items have a RGBA background. The video and sound (aside) backgrounds are HSLA. Use of CSS3 styles for color(RGBA & HSLA)
  4. The navigation menu will show you all of the link states. It also demonstrates the last-of-type as the reference item is wider than its siblings using it. The about page targets the first image using first-of-type to modify it. Use of dynamic pseudo-classes to highlight hyperlinks states of link, visited, active, hover; focus and two structural pseudo-classes: first-of-type and last-of-type
  5. The website background image uses background-size, the services cards use padding, margin, and border requirements. The support page utilizes the background-clip and background-origin. Use of CSS3 background styles (background size, background-clip, background-origin, padding, margins, borders, use of an appropriate border styles, use of rounded corners)
  6. The photography challenge first letter is a drop cap. Your home page should have a drop cap created with CSS3.
  7. The flex items and containers are defined with classes. The tooltips, slideshow dots container, and the dark mode and marking button use ids as well. Define document content with the class and id properties and create styles for them.
  8. Use each of the following page layout techniques:
    1. The gallery uses flex layouts. Grid
    2. Both services and contact both use flex layouts. Flex
    3. The navigation menu uses float positioning. Float
    4. The About page uses positioning on its images! Positioning
  9. The links will bring you to elemnets that use text and box shadows, respectively. It's the main page header and the video. Use of CSS3 text and box shadows
  10. The images are rotated to various levels. Use of CSS3 transform rotate feature
  11. When on mobile, the bottom images have opacity applied for a unique see-through stacked effect. Use of CSS3 opacity applied to two images to appear semi-transparent

Web Form

  1. Contact form uses the telephone and email inputs along with the normal text inputs and a textarea. Additionally a submit button. Use of appropriate form elements including some of the new HTML5 form elements

Responsive Design

  1. Try resizing any of the website pages and they'll work for desktop, mobile, and tablets! Positioning is used throughout all of the webpages (including these tooltips!) Your website design should include responsive design implementation using concepts covered in this course that would accommodate desktop, tablet, and mobile designs including positioning, grid, and flex layouts.
  2. If you go below 480px, a navicon menu appears which shows submenus using contextual selectors. The use of contextual selectors to control sub level menus is required, including a navicon image for mobile design.
  3. The gallery images are responsive, collapsing the columns as the screen size shrinks and expanding them as the screen grows. Images should also be responsive for different screen sizes and devices.
